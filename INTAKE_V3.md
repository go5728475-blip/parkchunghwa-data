## II. **추출 ** **구조**

```
[메뉴 A~E]
      │
      ▼
[C 유형 후보 좁히기] ──→ D ──→ S
      │                      │
      └──────────┬───────────┘
                 ▼
           B (후보) + U (축)
                 │
                 ▼
        [5필드 완료 §V] → 분선언 → 운 → 직답
```

### **II-1. ** **스키마**

```yaml
intake_v3:
  menu: A | B | C | D | E
  D: string                        # Decision · 1문장
  S:
    structure: string
    timing: string
    constraint: string | null
  B:                               # Bun · ≥2
    - { domain: 관계|배우자|진로|직업|학습|재물, priority: 1|2|3 }
  U:                               # Luck · 1~3
    - { axis: string, linked_to: D | S | C }
  C:                               # Conflict · 필수 1종
    type_id: string                # §III 코드 (A-C1 등)
    layer: L1 | L1×L2 | L2
    axis_a: string                 # 충돌 한쪽 (구조·갈래)
    axis_b: string                 # 충돌 반대쪽
    linked_B: [domain]             # 이 충돌을 풀 分
  raw_utterance: string            # 보조 · 추출용 ✗
```
