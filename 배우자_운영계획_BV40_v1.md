## 2. 3층 구조

```
[고객 질문]
     ↓
L1 판별축 (BV-37) — "무엇이 해당되는가"
     ↓
```

### 3-2. 9축 읽기 순서

1. **축 9** — 연월일시 위치 (무대·시기)

### 3-3. 항목 매칭 규칙

- 사주에서 **실제 해당**하는 항목만 L2·L3 조회.
- **전 항목 나열 금지.** 한 사주당 **강한 항목 2~4개**만 운영 출력.
- `미등재` 항목 → **임의 해석 금지**, 다른 축으로 대체하지 않음.
- `원문 약함` / `추가 축 필요` → L3에 그대로 반영 (「이 축만으로는 알기 어렵다」).

## 8. BV-40-A — 가상 고객 반응 예측

**목적:** **실제 검증 아님** · 답변 투입 후 **가상 고객 반응 예측**만 수집

### 수집 항목 (3항)

```
① 예상 기억 문장:

② 예상 「어?」 / 「그게 무슨 뜻?」 발생 부분:

③ 예상 행동 (Y / PARTIAL / N):
```

### 기록 시 함께 적을 수 있는 것 *(선택)*

- 분 유무 (없음 / 관계·배우자 分 / 직업의 分 등)
- 해당 **분 관련 문장** (원문 보존)

### 금지

- **정답 판정** · **맞다/틀리다** 판정
- **통계 결론** · **실고객 효과 단정**
- PASS/FAIL을 **실측 결과**처럼 해석
- 구조 수정 · 새 엔진 · 새 규칙 생성

### #10 관측 수정 *(FAIL 원인 재기록)*

| | 내용 |
|---|------|
| **FAIL ✗** | 공간적·시간적·역할적 **설명 자체** |
| **FAIL ✓** | **분 용어 남발** → 이론 설명처럼 들림 → 몰입도 저하 |
| **유지 가능** | 공간적 / 시간적 / 역할적 **설명 내용** |
| **문제 패턴** | 문장 대부분 「○○分」→ 「그래서 내 상황이 어떤데?」 상실 |

---

### 강화 가설 *(미확정 · 기록만)*

**기존 (약함)**

```
分 → 현재 운 → 현상 설명 → 판단
```

**수정**

```
分 → 현재 운이 그 分에 어떤 영향을 주는가 → 현상 설명 → 판단
```

**이유:** 현재 운만 설명하면 일반 운세 — 「외로움 운」「이성운」 등 **누구에게나** 가능 → 반응 약함.

**고객이 듣고 싶은 것 (예시 구조)**

> 당신의 관계의 分은 ○○입니다.  
> 그런데 현재 운에서는 그 分이 **약해지거나 / 흔들리거나 / 반대로 활성화**됩니다.  
> 그래서 지금 이런 일이 생기는 것입니다.

**관측 수정**

| | 반응 |
|---|------|
| 운 **자체** | 약함 |
| **分 없는** 운 설명 | 약함 |
| **分 없는** 행동 조언 | 약함 |
| **分 기준** + 현재 운이 **무엇을 흔들·키우·시험**하는지 | 반응 |

**현재 강화 문장**

> 운은 단독으로 존재하지 않는다. 운은 항상 分에 작용한다.  
> 고객은 운을 듣고 싶은 것이 아니라 **내 分에 지금 어떤 일이 벌어지고 있는지**를 듣고 싶어한다.

**운영 주의 (가설):** 分 = **판단 근거** · 分 = **주인공 ✗** · **분 용어 남발 ✗**
